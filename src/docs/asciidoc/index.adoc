FaithLog의 상세 API 계약은 Spring REST Docs 테스트가 검증한 request/response snippets를 기준으로 문서화한다.
Swagger/springdoc은 간단한 API 탐색용으로 유지하지만, 정확한 계약 문서는 이 REST Docs 산출물을 기준으로 본다.

== Common Response Envelope

모든 API 응답은 현재 공통 `ApiResponse` envelope를 사용한다.

|===
|Field |Description

|`success`
|요청 성공 여부

|`code`
|공통 응답 코드

|`message`
|응답 메시지

|`data`
|응답 데이터. 실패 응답에서는 생략될 수 있다.

|`timestamp`
|응답 생성 시각
|===

=== Login

`POST /api/v1/auth/login`

로그인 성공 시 토큰은 JSON response body로 반환한다.
모바일 클라이언트는 `accessToken`을 `Authorization: Bearer {accessToken}` 인증에 사용하고, `refreshToken`은 Keychain, Keystore, SecureStore 같은 플랫폼 보안 저장소에 저장한다.

토큰 만료 기준:

|===
|Field |Value

|`accessTokenExpiresIn`
|`1800`

|`refreshTokenExpiresIn`
|`1209600`

|`tokenType`
|`Bearer`
|===

=== Get My Weekly Devotion

`GET /api/v1/campuses/{campusId}/devotions/me/weeks/{weekStartDate}`

현재 로그인 사용자의 특정 주차 경건생활 기록을 조회한다. 응답의 `dailyChecks`는 항상 월요일부터 일요일까지 7일치다. 해당 주차 row가 아직 없으면 404를 반환하지 않고 7일치 기본 응답을 반환한다. 기본 응답은 체크값이 모두 false이고, 요약 count와 `saturdayLateMinutes`는 0이며, `weeklyRecordId`, `submittedAt`, `dailyChecks[].id`는 null이다. 주간 row가 이미 있어도 저장되지 않은 날짜는 `id = null`, 체크값 false로 합성한다. 조회 API는 기본/누락 날짜 응답을 만들기 위해 DB row를 생성하지 않는다.
