FaithLog의 상세 API 계약은 Spring REST Docs 테스트가 검증한 request/response snippets를 기준으로 문서화한다.
Swagger/springdoc은 간단한 API 탐색용으로 유지하지만, 정확한 계약 문서는 이 REST Docs 산출물을 기준으로 본다.

== Common Response Envelope

모든 API 응답은 현재 공통 `ApiResponse` envelope를 사용한다.

|===
|Field |Description

|`success`
|요청 성공 여부

|`code`
|공통 응답 코드

|`message`
|응답 메시지

|`data`
|응답 데이터. 실패 응답에서는 생략될 수 있다.

|`timestamp`
|응답 생성 시각
|===

=== Login

`POST /api/v1/auth/login`

로그인 성공 시 토큰은 JSON response body로 반환한다.
모바일 클라이언트는 `accessToken`을 `Authorization: Bearer {accessToken}` 인증에 사용하고, `refreshToken`은 Keychain, Keystore, SecureStore 같은 플랫폼 보안 저장소에 저장한다.

토큰 만료 기준:

|===
|Field |Value

|`accessTokenExpiresIn`
|`1800`

|`refreshTokenExpiresIn`
|`1209600`

|`tokenType`
|`Bearer`
|===

=== Change Charge Status As Admin

`PATCH /api/v1/admin/charges/{chargeItemId}/status`

캠퍼스 관리자 또는 전역 `ADMIN` 사용자는 `UNPAID` 상태 청구를 `PAID` / `WAIVED` / `CANCELED` 상태로 변경하거나, 종료된 청구를 `UNPAID` 상태로 되돌릴 수 있다. 관리자 `UNPAID -> PAID` 전환은 서버 현재 시각을 `paidAt` 필드에 저장하고, terminal 상태에서 `PAID` 상태로의 전환은 409로 실패한다. `PAID` 상태 청구를 `UNPAID` 상태로 되돌릴 때는 `paidAt` 필드를 비운다.

`PENALTY + DEVOTION_RECORD` 청구를 `UNPAID -> CANCELED`로 변경하면 같은 transaction에서 source weekly record의 `submittedAt`을 null로 재오픈한다. source weekly record는 청구와 같은 campus/user여야 하며, 기존 daily checks는 삭제하지 않는다. `WAIVED`, `COFFEE`, `POLL_RESPONSE` 취소는 경건생활을 재오픈하지 않는다. 재제출 벌금이 양수이면 기존 CANCELED source charge row를 UNPAID로 재사용해 현재 금액과 계좌 snapshot을 갱신하고, 0원이면 CANCELED row를 유지하며 새 row를 만들지 않는다.

== Meal Duty, Poll, and Post-settlement Billing

MEAL 운영 API는 같은 캠퍼스의 현재 ACTIVE MEAL 담당자만 사용할 수 있다. service ADMIN 또는 캠퍼스 관리자도 MEAL duty가 없으면 `403`이다. 일반 사용자의 투표 목록·상세·응답과 사용자 옵션 추가는 기존 캠퍼스 멤버 정책을 유지한다.

=== Manage and Close Meal Poll

관리 목록은 MEAL poll만 pagination하며 마감 후 3일이 지난 CLOSED poll도 반환한다. `settlementStatus`는 `NOT_CHARGED` 또는 `CHARGED`다. 수동 종료는 OPEN MEAL poll의 상태만 CLOSED로 바꾸며 기존 `endsAt`을 유지하고 settlement나 charge를 만들지 않는다. 전용 종료 API에서 다른 캠퍼스와 non-MEAL poll은 모두 `404 POLL_NOT_FOUND`로 숨기며, 일반 관리자 종료 API로도 MEAL poll을 우회할 수 없다.

=== Create Poll-level Meal Charges

요청의 `paymentAccountId` 하나가 모든 option group에 공통으로 적용된다. 응답자가 있는 모든 option을 정확히 한 번 포함해야 하며, 0명인 option은 제외한다. `PER_MEMBER`는 입력한 1인당 금액을 곱하고, `GROUP_TOTAL`은 `ceil(enteredAmount / responseCount)`를 정수 연산으로 계산한다. `requestedTotalAmount`, `actualTotalAmount`, `roundingAdjustment`를 분리한다. 전체 settlement/group/charge는 단일 transaction이며 같은 poll 재청구는 `409`다.
